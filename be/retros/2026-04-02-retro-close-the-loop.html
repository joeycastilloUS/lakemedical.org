Boards 27–30 · 68 pts · 16 items · lakemedical.org NOUS Wire Auth

What Went Well

  • Wire protocol is solid. NTRP0001 binary format, pack/unpack in C and JS, triple-based commands — all worked on first try across all layers.
  • Double-hop encryption works. Cockpit key (client ↔ relay) + transport key (relay ↔ prime) — two different HMAC-derived keys, AES-256-GCM on both hops. Zero plaintext on the wire.
  • Cloudflare Worker TCP connect. The connect() API handles raw TCP beautifully. The Worker is a clean 160-line proxy with zero state.
  • catalog_server compiles and runs. The full NOUS C stack — memory, security, comms, intelligence skills — builds and runs on GCE with one gcc command.
  • Session minting on relay. 64-hex token, IP-bound, 900s TTL, per-user limits. Security-first design proved out in testing (rate limit caught double-auth).

What Didn't Go Well

  • Prime selection was a silent killer. find_first_prime() returns the first DNS-discovered prime. With two regions in DNS (asia-east1 + us-west2), the wrong-region prime was selected. Transport key mismatch caused silent failure — relay just returned "prime unreachable" with no indication of which prime was tried or why.
  • TRACEF vs printf confusion. All diagnostic output uses TRACEF (ring buffer, port 9090) not printf (journal). When debugging via journalctl, critical information was invisible. stdbuf -oL fixed printf buffering, but the real diagnostic data was always on the trace port.
  • DNS verification is signature-only. The relay verifies DNS TXT records by checking ed25519 signatures, not TCP reachability. Stopped instances still "verify" because the signed record persists. This is by design (crypto trust, not network trust) but surprising when debugging.
  • startup.sh was incomplete. Missing include paths for qr.h, execute.h, translate.h. The compile failed silently until manually inspected. The build script should error loudly.

Lessons

  • find_first_prime() needs region awareness. The relay knows its own region (NOUS_NODE_REGION). It should prefer primes in the same region, or at minimum try multiple primes before giving up. One prime failure should not be fatal.
  • Add printf to relay_wire_to_prime(). A single printf of the target host:port would have cut debugging time by 80%. The TRACEF-only approach is clean for production but brutal for initial deployment.
  • DNS TXT records = persistent infrastructure. Treat DNS prime entries like server deployments. Don't leave stale entries for stopped instances. The asia-east1 entry should have been removed when that prime was mothballed.
  • Test from outside the VPC. Initial tests from localhost on the relay worked (internal routing). The real failure path (external IP + DNS primes) only showed up when testing from outside.

Recommendations

For NOUS Core

  • Add relay_wire_to_prime() printf: "relay: forwarding to %s:%d\n"
  • Implement prime fallback: try next prime if first returns error or timeout
  • Add region-preference to find_first_prime(): same-region primes first
  • Add --no-dns flag to relay for controlled deployments
