Test 5 — Expected Failure Analysis

Test 5 sends a second auth request within the same TOTP window. The relay enforces a per-user session limit — creating a second session for the same user returns (resp, status, error) + (resp, message, session limit).

This is correct security behavior, not a bug. The browser E2E test (e2e-auth.html) reuses the token from Test 4 for the session check, avoiding this.
