~P: Fix QR Code Encoder

Port proven C qr.c to JS · Replace broken browser encoder · lakemedical.org

Plan — Awaiting Approval

Intent

Why: Users cannot register on lakemedical.org because the QR code displayed after registration is unscannable. They can’t set up their TOTP authenticator app.

Purpose: Replace the broken JS QR encoder with a faithful port of the proven C implementation (qr.c, 606 lines, 200+ test assertions).

What: A new qr.js that generates identical output to qr.c for any input.

Scope

In Scope

  • Port C:\metal\auth\qr.c to JavaScript (line-by-line, algorithm-for-algorithm)
  • GF(256) arithmetic, Reed-Solomon, bit-stream encoding
  • Multi-block interleaving (the #1 fatal bug)
  • Correct format info placement (the #2 fatal bug)
  • All 4 penalty rules for mask selection (the #3 fatal bug)
  • Same API: qr_encode(text) → {data: Uint8Array, size: number}
  • Versions 1–6 (matches C), EC level M (what otpauth needs)
  • Verify with real otpauth URI against phone scanner

Out of Scope

  • Numeric/alphanumeric encoding modes (byte mode only, like C)
  • Versions 7–40 (V6 = 41×41 modules, handles 108 bytes at EC M — plenty for otpauth)
  • Terminal rendering (browser uses canvas, not half-blocks)
  • Changes to auth-gate.js or nous.js (they already expect the right API)

Approach

Phase 1: Port qr.c → qr.js

Direct translation of every function from C to JavaScript. No creative interpretation — match the C exactly.

Phase 2: Replace & Test

  1. Replace alumni/js/utils/qr.js with the port
  2. No changes to auth-gate.js — it already calls qr_encode(uri) and uses {data, size}
  3. Deploy to Cloudflare Pages
  4. Register a test user in the browser, scan the QR with a phone

Constraints

  • Vanilla JS only — no frameworks, no build tools, no imports
  • Must expose window.qr_encode (existing API contract)
  • Byte mode only, EC level M (hardcoded, matches the C caller on the prime)
  • Versions 1–6 only (max 108 bytes at EC M — typical otpauth URI is ~75 bytes)
  • Tables must match C exactly — data_codewords, ec_codewords_per_block, ec_blocks, align_pos
