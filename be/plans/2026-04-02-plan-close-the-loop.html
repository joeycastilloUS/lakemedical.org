Approach

Phase 1 — Diagnose the 502

The Worker sends a 0xCA envelope to relay:8080. The relay decrypts with cockpit key, dispatches to handle_wire_auth, which calls relay_wire_to_prime(). The response comes back and the relay encrypts it with cockpit key. The Worker tries to decrypt. Something breaks.

  1. SSH into relay — check journalctl -u nous-relay for the auth dispatch logs. Does it say [EW] wire request? Does it reach handle_wire_auth?
  2. Check prime reachability — can the relay TCP connect to the CLI primes (35.236.118.2:8080, 34.81.178.230:8080)? Are they running?
  3. Check PSK match — relay's NOUS_PSK env var (from systemd) must match the Worker's NOUS_PSK secret. Same hex string → same cockpit key.
  4. Check response format — if prime is unreachable, relay_wire_to_prime() returns an error response. Is that response properly encrypted with cockpit key before sending back to the Worker?

Most likely root cause: Prime is not running, relay_wire_to_prime() fails, and the error response either (a) isn't encrypted, or (b) is encrypted with the wrong key.

Phase 2 — Prime Readiness

  1. Check prime VMs — are they RUNNING in GCE? Start if TERMINATED.
  2. Check prime binary — does it have wire auth handling? Check a-d-d spec for prime's expected interface.
  3. Provision test user — register one TOTP user on the prime's user store. This requires understanding how the prime stores users (triple store? users.dat? TOTP seed file?).
  4. Verify relay→prime connectivity — from relay VM, can we TCP connect to prime:8080 and send a test wire auth frame?

Phase 3 — Full Round-Trip

  1. Test auth flow manually — send auth triples (user + TOTP code) via the E2E test script to lakemedical.org/api/wire. Expect: status 200 + NTRP response with session token.
  2. Test in browser — open lakemedical.org, use the auth gate login form with the test user. Expect: QR shows on register, TOTP validates on login, session timer starts, gate drops, app loads.
  3. Update E2E test — add real auth assertions (200 + valid NTRP response + session token present).
