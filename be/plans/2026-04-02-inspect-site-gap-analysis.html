Where lakemedical.org Is Behind

lakemedical.org is production-ready but the app itself (app.html) is essentially a copy of Alumni-Artifacts with minimal evolution. The gaps are in data depth and visualization richness:

  • No real alumni data loaded. The static JSON in data/alumni.json is sample/demo data. The enrichment pipeline exists but hasn't been run against a real registrar CSV yet.
  • No KPI value boxes. The NIEHS dashboard opens with instant stats (total records, coverage %, years of data). lakemedical.org's dashboard has charts but no at-a-glance metrics.
  • No geographic visualization. The NIEHS dashboard has choropleth maps showing alumni by state. lakemedical.org has practice_city/practice_state data from NPI but doesn't visualize it on a map.
  • No trend analysis. No multi-year comparison or cohort tracking. The data model supports class_year but it's not sliced by time period.
  • No Sankey/flow diagrams. Career path visualization (specialty → practice type → location) would be powerful with NPI data.
  • Dashboard is basic. Chart.js bar/pie charts. No interactive drill-down, no resizable plots, no export capability.
  • No bulk import UI. The enrichment pipeline (enrich.mjs) is CLI-only. No browser-based CSV upload for the admin user.
  • The app.html branding is half-done. The sidebar still references WesternU styles. The AcmeDev rebrand only covers landing + login pages, not the inner app chrome.

What's Worth Stealing from NIEHS Dashboard

  • KPI value boxes — Top of each section. Total alumni, active practitioners, published researchers, geographic reach. Instant visual summary.
  • Contextual sidebar filters — Filters change based on active view. Queue gets priority + date. Directory gets specialty + location. Dashboard gets time period.
  • State-level heatmap — We already have practice_state from NPI. A US map colored by alumni density would be high-impact for Lisa.
  • Career flow diagrams — Sankey showing class_year → specialty → practice location. Shows where graduates end up.
  • Time-period bucketing — Group by 5-year graduation cohorts. Compare engagement across eras.
  • Pre-computed summaries — Build aggregates at data load time, not on each render. Already possible with vanilla JS.

All implementable in vanilla JS + Chart.js. No R/Shiny needed.
